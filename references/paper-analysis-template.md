# 论文分析模板

用户可见回答和 Notion 笔记都使用此结构。默认使用简体中文；论文标题、模型名、数据集名、benchmark 名称、指标名和官方术语可保留原文，以保证准确性。

最终内容只保留标题和以下六个部分。不要加入元信息、作者、会议年份、来源链接、代码链接、本地 PDF 路径、上传状态、单独的局限章节或一屏总结，除非用户额外要求。

## 输出格式

Notion 正文必须按此格式组织：

```markdown
## <论文标题>

**TL;DR**

...

**论文 motivation**

...

**方法创新点**

...

**实验 benchmark**

...

**比较的 baseline**

...

**实验结论**

...
```

不要把六个 section 写成 `##` heading；它们必须是加粗文本标签。

## TL;DR

用 3-5 句话说明：论文解决的问题、核心方法、最关键实验结果、主要结论或主要 caveat。

## 论文 motivation

说明：

- 论文要解决什么问题。
- 现有方法为什么不够好。
- 这个问题在科学上或实际应用中为什么重要。
- 作者基于什么观察、假设或痛点提出该方法。

## 方法创新点

从两个层次描述方法：

- 整体流程：输入、核心模块、训练 / 推理流程、输出。
- 创新点：用 3-6 条说明相对已有方法的新意。

每个创新点都要写清楚：它想带来什么收益，以及引入了什么代价、假设或限制。

## 实验 benchmark

列出每个 benchmark 或数据集：

- 任务：
- 数据集 / benchmark 名称：
- 评价指标：
- 数据划分 / 实验设置：
- 主要结果所在位置，例如表格或图编号：

## 比较的 baseline

必要时按类型组织 baseline，例如：

- 传统 / 非学习方法
- 早期神经网络方法
- 同期强模型
- 本文方法的 ablation 变体

对最重要的比较，记录论文报告的具体数值，并说明提升是绝对提升、相对提升、是否做了统计检验，还是只属于定性比较。

## 实验结论

解释实验真正支持了什么：

- 主性能结论：
- 泛化能力结论：
- 效率 / 成本结论：
- 消融实验结论：
- 鲁棒性或失败案例结论：
- 仍未证明的部分：
